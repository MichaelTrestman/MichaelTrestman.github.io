Css Concepts

Margin, border, and padding.

Together, these three attributes make up the 'box model', which is a big part of how CSS determines the way DOM elements take up space in the browser window.
the box model

Content

What is the basic difference? The border is the boundary between the element and it's surrounding DOM environment. In the diagram, the div with class "box" has a purple background and a white border. The white border seperates the purple background of the "box" div from the black background of the surrounding "box-container" div.
Padding seperates the content (in this case, the word "content"), from the border. More padding, more space inside the div--in this case, more purple.
Margin seperates the element from other elements in the containing div. In this case, that would be between the white boundary whatever else is floating around in the DOM with the 'box' div.

Classes vs. Ids: best practices

When should you use a class? Classes can be applied many times, and a single DOM element can have many classes. This is powerful for giving structure to the visual appearance of page, and can allow you to quickly establish a coherent style website with lots of content and many pages. Partial overlap of the style classes between html elements (i.e. some but not all classes shared), can be used to establish visual hierarchy and grouping.

When should you use an ID? One element may only have one ID, and an ID may only apply to a single element within an HTML documents. This limits their utility for styling; in general, classes give you more flexibility for the reasons above. However, IDs can be used to style elements that are unique within a page (but may appear on multiple pages within a site), such as a header, nav bar or footer.
IDs also allow you to anchor links within a page, and are very useful for that reason.

 Why is this one of your favorite sites? Well, I didn't know about this website until today, but the other sites I picked to blog about, I picked because of their content/function, without much regard for design aspects (except in a very functional sense). I discovered howlt.com via awwwards.com's list of best webistes. Really, this site breaks a lot of my general rules for what i consider good websites: I generally think bells and whistles should be kept to a minimum, and this website is basically just a collection of bells and whistles (it even starts playing music automatically on page-load, which seems like a terrible idea generally). However, it is a SUPER skillfully put together collection of bells and whistles, and it comes together for a really fun experience. navigating the website is like playing a game, and that is a really powerful thing to achieve. I also just think I can learn a lot of cool techniques by exploring the css and javascript for the site.
